# STM32H750双ADC相位差测量

先从`STATUS.md`读取当前引脚、时钟、量程和风险，再核对`Core/Src/adc.c`、`Core/Src/tim.c`、`FML/ADC_FML.*`、`BLL/ADC_BLL.*`、`BLL/PHASE_BLL.*`和`API/PHASE_API.*`。仅实现以下测量链路。

## 组织数据链路

1. 把ADC1作为主ADC、ADC2作为规则同步从ADC，用TIM TRGO同时触发。
2. 先在独立模式下分别校准ADC1/ADC2，再切换到双重规则同步模式。
3. 用公共CDR和32 bit Normal DMA采集配对样本，按当前数据格式拆分两路ADC码值。
4. 使DMA缓冲区32 byte对齐并放在`.dma_buffer`；DMA前清理/失效DCache，CPU读取前再失效DCache。
5. 通过NDTR轮询单帧完成；NDTR归零后停止TIM、停止DMA并处理数据，不要让DMA回调与轮询同时重启采集。

## 先测公共频率

1. 以标称1 MS/s同步采集4096组数据。
2. 对两路分别去直流，用ADC1的Hann窗4096点FFT定位主峰。
3. 在主峰附近搜索，最大化两路正弦拟合的解释功率，得到未取整公共频率。
4. 保留未取整频率用于后续相干配置和拟合，以吸收MCU采样时钟的比例误差。
5. 若题目保证输入为整数kHz，另计算：

   ```text
   nominal_frequency = floor(measured_frequency / 1000 + 0.5) * 1000
   ```

   仅用标称频率显示、判断量程和查询校准表。

## 相干采样测相位

1. 在允许采样率内联合搜索TIM整数分频、帧长和整数周期数。
2. 最小化窗口闭合误差：

   ```text
   closure_error_deg = (frequency * sample_count / sample_rate - cycles) * 360
   ```

3. 用未取整频率对两路做带独立DC项的基波最小二乘拟合。
4. 同时拟合三次谐波；两路三次谐波/基波幅度比都达到阈值时，转入三角波/强谐波路径。
5. 正弦路径使用两路基波拟合相位之差。
6. 三角波路径在一个周期内搜索归一化互相关峰，以基波相位作为时延搜索先验，再用黄金分割细化到小数采样点：

   ```text
   triangle_phase = -lag_samples * 360 / samples_per_period
   ```

7. 统一计算并包装到`(-180°, 180°]`：

   ```text
   phase = ADC2_phase - ADC1_phase - calibration(nominal_frequency)
   ```

8. 输出后立即返回测频阶段，不长期复用旧频率。

## 判定和输出有效帧

仅在以下项目均达标时输出：

- 标称频率在当前量程内。
- 两路基波幅度达到最小输入阈值。
- 正弦路径的两路拟合质量达标。
- 三角波路径的基波拟合质量和最佳互相关均达标。
- 窗口闭合误差达标。

无效帧不输出相位。正常结果只保留：

```text
freq=10000.00Hz phase=-90.123deg
```

## 验证功能

1. 先用同源一分二验证0°和通道固定延迟。
2. 对正弦和三角波分别覆盖量程内关键频点及0°、±90°、接近±180°。
3. 每个点统计连续有效帧的平均误差、最大误差、标准差和丢帧率。
4. 仅在实物验收通过后更新校准表和精度结论。
